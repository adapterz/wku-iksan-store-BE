## 5. 추가 작업사항

### 5.1 API 설계 Wiki 업로드
- 기존 논의 내용에 본인 의견을 종합하여 API 설계 문서를 Wiki에 업로드
- 누락된 기능이나 잘못된 코드 발견 시 코멘트 요청

### 5.2 중복 API 삭제
- **삭제 대상**: 결제 확인, 결제 완료 확인, 선물 사용
- **사유**
  - 결제 확인 / 결제 완료 확인: 둘 다 주문의 결제 상태 확인 기능으로, 주문 상세 조회에서 함께 제공 가능하여 별도 분리 불필요
  - 선물 사용: 바코드 사용 시 곧 선물 사용으로 처리되는 기능이므로 목적·결과가 동일하여 하나로 통합
- **결론**: 주문 상세 조회 + 바코드 사용 기능만으로 현재 요구사항 충족 가능

### 5.3 ERD 설계도 Wiki 업로드 (M2 한정)
- M2에 실제 적용될 ERD 설계도 업로드
- **요청 사항**
  - 각 테이블의 역할과 흐름, 하단 '의사결정 내역' 숙지
  - 이해되지 않는 부분은 AI 활용하여 학습
  - 기획과 어긋나는 부분이나 예외 상황 발견 시 자유롭게 코멘트

### 5.4 View 문서 수정
- View2 내 "확정된 필요 api를 AI 활용하여 api 기능 설계" 문구 → "API 기능 목록 확정"으로 간소화
- 개인 폴더 구조를 닉네임 기반 → 직무별로 재구성
- View의 금일 할 일 및 진행 내용 갱신

### 5.5 API 추가/삭제 (로그아웃, 주문 취소)
- **추가**: 로그아웃 API
  - M2 '로그인·회원가입 완성' 단계에서 세션 및 토큰 파기를 위해 필수이나 기존 명세서에서 누락되어 있어 추가
- **삭제**: 주문 취소 API
  - 환불 처리 기능이나 M2의 정방향 핵심 흐름(주문 → 결제 → 선물함 → 사용)에 포함되지 않는 오버스펙으로 판단하여 이번 마일스톤에서 제외
- **요약**: 필수 기능(로그아웃)은 추가하고 불필요한 기능(주문 취소)은 삭제하여, M2 목표 달성에 꼭 필요한 기능만 빠르게 구현
