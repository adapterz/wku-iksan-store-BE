# 홈 화면 - 구현 API 선별

## 구현 O (API 필요, 1개)

| 번호 | 항목 | 이유 |
|------|------|------|
| 1 | 상품 목록 조회 (GET) | 홈 화면에서 상품 이미지, 상품명, 브랜드명, 가격 등을 표시하기 위해 서버에서 상품 목록을 조회해야 함 |

---

## 구현 X (API 불필요, FE/UI 처리, 9개)

| 번호 | 항목 | 이유 |
|------|------|------|
| 2 | 로그인 버튼 클릭 | 로그인 페이지로 이동만 수행하므로 API 호출 불필요 |
| 3 | 상품 상세 페이지 이동 | 선택한 상품의 ID만 전달하여 페이지 이동하면 됨 |
| 4 | 선물함 버튼 클릭 | 선물함 페이지로 이동만 수행 |
| 5 | 검색 버튼 | 검색 아이콘 및 화면 이동 UI만 구현 |
| 6 | 카테고리 메뉴 | 버튼 UI만 구현, 필터 API 사용 안 함 |
| 7 | 이벤트 배너 | 정적 이미지(UI)만 표시 |
| 8 | 베스트/추천 상품 영역 | UI만 구현, 별도 조회 API 사용 안 함 |
| 9 | 찜(좋아요) 아이콘 및 개수 | UI만 구현, 찜 API 사용 안 함 |
| 10 | 하단 탭바(Home / 선물함 / 선물하기) | 화면 이동 UI만 구현 |

---

## 결론

홈 화면은 대부분이 화면 이동과 UI 구성으로 이루어져 있으며, 서버와 통신이 필요한 기능은 **상품 목록 조회(GET)** 하나뿐이다. 나머지 기능은 M2 범위에서는 UI 또는 페이지 이동만 구현하고, 검색·카테고리·베스트·찜 등의 기능은 API를 구현하지 않는다.
