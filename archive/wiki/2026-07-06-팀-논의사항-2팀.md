# 🚀 웹 서비스 배포 파이프라인 개념 정리

## 📌 개요

AWS EC2(Ubuntu) 환경에 **Node.js 기반 웹 애플리케이션(Frontend / Backend)** 과 **MySQL 데이터베이스**를 구축하고, **PM2**를 활용하여 서비스를 24시간 안정적으로 운영하는 전체 배포 과정을 정리함.

> 즉, 클라우드 서버를 생성하고 개발 환경을 구성한 뒤, 웹 서비스를 외부에서 접속 가능하도록 배포하는 과정임.

---

# 🛠️ 배포 구성 요소

### ☁️ AWS EC2 (Ubuntu)
- 클라우드 환경의 가상 서버
- 웹 애플리케이션이 실행되는 운영 환경 제공

### 🟢 Node.js
- JavaScript 런타임 환경
- Backend 및 Frontend 서버 실행

### 🗄️ MySQL
- 관계형 데이터베이스(RDBMS)
- 서비스 데이터를 저장 및 관리

### 🌱 Git / GitHub
- 프로젝트 버전 관리
- 서버에서 최신 소스코드 Clone

### ⚙️ PM2
- Node.js 프로세스 관리자
- 서버를 백그라운드에서 실행
- 장애 발생 시 자동 재시작 지원

---

# 📦 배포 진행 과정

## 1️⃣ EC2 서버 생성 및 초기 환경 구성

### 수행 작업

- AWS EC2 인스턴스 생성
- Ubuntu 운영체제 설치
- 보안 그룹(Security Group) 설정
    - TCP 3000
    - TCP 8080
    - TCP 3306
- Node.js 설치
- MySQL 설치
- MySQL 초기 보안 설정

---

## 2️⃣ MySQL 데이터베이스 구성

### 수행 작업

- 서비스 전용 계정 생성
- 사용자 권한 부여
- 원격 접속 허용
    - `bind-address` 설정
- 데이터베이스 생성
- 서비스에서 사용할 테이블 생성

예시

- User
- Post
- Comment
- Attachment

---

## 3️⃣ GitHub 프로젝트 배포

### 수행 작업

- GitHub Repository Clone
- Backend 프로젝트 다운로드
- Frontend 프로젝트 다운로드
- `npm install` 실행
- `.env` 환경 변수 파일 생성

환경 변수 예시

- DB_HOST
- DB_USER
- DB_PASSWORD
- PORT
- SECRET_KEY

---

## 4️⃣ 서버 실행 및 서비스 배포

### 수행 작업

- Node.js 서버 실행
- PM2를 이용한 백그라운드 실행
- 서비스 정상 동작 확인

접속 형태

```
http://EC2_Public_IP:8080
```

# 📌 핵심 용어 정리

### EC2 Instance
AWS에서 제공하는 가상의 컴퓨터(클라우드 서버)

### Ubuntu
EC2에서 사용하는 Linux 운영체제

### npm (Node Package Manager)
Node.js 패키지 관리 도구

### PM2
Node.js 프로세스 관리 도구

### .env
환경 변수 및 민감 정보를 관리하는 설정 파일

# 🖥️ 주요 명령어 정리

## 🐧 Linux(Ubuntu)

### `sudo su`

- Root 권한으로 전환

### `apt update -y && apt upgrade -y`

- 패키지 목록 갱신
- 설치된 패키지 최신 버전으로 업데이트

### `ll`

- 현재 디렉터리의 파일 및 권한 확인

### `mkdir`

- 새로운 디렉터리 생성

### `cd`

- 디렉터리 이동

### `systemctl restart`

- 특정 서비스 재시작

---

## 📝 vi 편집기

### `vi`

- 텍스트 편집기 실행

### `:`

- 명령 모드 진입

### `wq`

- 저장 후 종료

### `q!`

- 저장하지 않고 종료

---

## 🗄️ MySQL

### `mysql -u root -p`

- MySQL 접속

### `FLUSH PRIVILEGES;`

- 변경된 권한 즉시 반영

---

## ⚙️ Node.js / PM2

### `npm install`

- 프로젝트 의존성 패키지 설치

### `pm2 start`

- 서버를 백그라운드에서 실행

### `pm2 monit`

- 실행 중인 서버 상태 및 리소스 모니터링
