## 채팅 스레드 정리 

### 협업 방식
- API, ERD 확정은 가급적 오전 중 마무리
- 검토 방식: AI로 1차 확인 → 이해 안 되는 부분 학습 → 문제 되는 부분 코멘트

### 기능 관련 논의 (선물 조회)
- **받은 선물 조회**: 기존 기능 유지 확정
- **미사용/사용 구분**: 별도 기능화하지 않고, 받은 선물 조회 API의 `status` 값을 기준으로 **FE에서 탭으로 구분**하기로 결정
- 선물 바코드 상세조회는 조회 전용(포괄적 성격)이라 별도 처리 불필요

### M0-1 관련
- 팀원 전원 11:40까지 노션에 추가된 M0-1 문서 정독하기로 함
- M0-1은 기존 구조에서 테마만 입힌 것으로 판단
  → API 기능 재설계 불필요, **예시 데이터만 2팀 테마로 교체**
  → "상품"을 "유저"가 구매/선물하는 프로세스 자체는 동일하다는 데 합의

### 마일스톤 회의 누락
- M0 검토 없이 M0-1 작업을 먼저 진행
- 오후에 M0 피드백을 받아 팀 커뮤니티에 공유
- M0-1 작업 중단 후 M0 보완 작업으로 전환

### 팀 구성 / 역할
- 신규 합류 Chloe, 배정 직무: **BE**
- 기존 BE 역할의 부(副) 역할이 DB임을 확인
- **정BE + 부DB** 형태로 BE 역할 2명(Chloe, Aon) 체제 운영 결정
- 동일 직무 2인 체제는 협업 심화 기회이므로 소통 중요하다는 조언
