# 메시지 편집 페이지 - 구현 API 선별


## 구현 O (API 필요, 2개)

| 번호 | 항목 | 이유 |
|---|---|---|
| 3 | 기본 텍스트 조회 (GET) | "나는 내가 챙긴다 소중한 나에게 주는 선물"이라는 기본 텍스트를 서버에서 불러와 편집 진입 시 표시해야 함 |
| 4-확인 | 확인 버튼 클릭 시 메시지 저장 (PATCH/POST) | 입력한 텍스트(최대 100자)를 카드(주문)에 반영하려면 서버에 저장 요청 필요 |

---

## 구현 X (API 불필요, FE 상태로만 처리, 4개)

| 번호 | 항목 | 이유 |
|---|---|---|
| 4-포커스/커서 | 카드 포커스, 마지막 글자 옆 커서 깜빡임 | 화면 렌더링/UI 상태일 뿐, 서버 데이터와 무관 |
| 4-글자수 | 최대 100자 입력, 글자 수에 따른 크기 변경 + 박스 내 전체 표시 | 입력 제한 및 스타일 변경은 CSS/JS로 처리되는 클라이언트 로직 |
| 4-삭제 | 텍스트 모두 삭제 | 입력창 내부 상태(state) 초기화일 뿐 |
| 4-취소 | 좌측 상단 취소 버튼 클릭 시 원래 페이지로 복귀 | 저장 없이 페이지 이동만 하므로 API 호출 불필요 |

---

## 결론
메시지 편집 페이지는 **글자 크기 변경, 커서 표시, 텍스트 삭제 등 대부분이 프론트엔드 상태(state)로 처리** 가능하고, **서버와 통신이 필요한 항목은 "기본 텍스트 조회"와 "확인 시 메시지 저장" 2개뿐**이다.
