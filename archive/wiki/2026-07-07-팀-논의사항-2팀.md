## 🔧 공지 및 Git 관련 사항

- 어제자 채팅 스레드에 활동이 적어 배포 교육 학습 내용을 기반으로 문서화 진행
- 각자 담당 작업은 실시간으로 view 문서에 업데이트할 것
- `develop` 브랜치에서 `.env` 파일 제거 → **git pull 필수**
  - `main`은 `develop` 머지 시 자동 반영되므로 별도 조치 없음
- FE 기본 가이드에 따라 JS 파일명 수정 → git pull 필요
- **[구조 변경]** 정적 서빙 최적화를 위해 `server` 폴더 삭제 및 최상위 루트로 이동 (develop 반영 완료)
  - 사유: React 등 프레임워크 빌드 없이 바닐라 JS로 `public` 폴더 정적 서빙만 하므로, 불필요한 폴더 구조/의존성 격리 제거 및 경로 문제 예방 목적

### ⚠️ 브랜치 전략 이슈 및 정정
- 초기에 브랜치 전략을 잘못 이해하여 `develop`에서 직접 작업 진행됨
- 올바른 방식: `develop` 기준으로 `feature/...` 브랜치 생성 → 작업 후 PR → `develop`으로 머지
- **Merge는 각자 진행하지 않고, PR 검토 후 관리자(Aon) 1인이 일괄 진행**
- 기존 작업 내용은 문제 없다고 판단, 별도 롤백 없이 향후부터 Git Flow 준수하기로 결정
- Aon이 팀원들에게 브랜치 전략 인수인계 예정 (브랜치 분리 방법, PR/머지 담당자 등)
