# 로그인 기능(Login) 기능 분류

> 전체 프로젝트 흐름은 아래 2단계로 한정됨(범위 기준)
> **[1] 로그인 → [2] 회원가입 및 계정 생성**

---

## ✅ 구현 필요 기능 (API 연동 필요)

| 번호 | 기능 | 해당 단계 | 비고 |
|---|---|---|---|
| 1 | 로그인 인증 처리 | [1] | 아이디/비밀번호 기반 사용자 인증 |
| 2 | 회원가입 | [2] | 신규 사용자 계정 생성 및 DB 저장 |

---

## ❌ API 구현 불필요 기능 및 근거

| 번호 | 기능 | 근거 | 담당 |
|---|---|---|---|
| 3 | 이용약관 | 실제 약관 페이지가 존재하지 않는 더미 UI 요소이며 서버 데이터 처리 없음 | **FE** |
| 4 | 개인정보 처리방침 | 정적 UI 요소로 서버 통신 및 데이터 처리 없음 | **FE** |
| 5 | 고객센터 | 현재 기능 미구현 상태이며 단순 UI 요소로만 존재 | **FE** |
| 6 | 카카오 로그인 버튼 | 실제 OAuth 연동 없이 UI만 구현하는 더미 버튼이며 인증은 일반 로그인으로 처리 | **FE** |

---

## 📌 결론

- 로그인 기능은 **“사용자 인증 + 계정 생성(회원가입)”** 영역만 API 구현 대상이다.
- 이용약관, 개인정보 처리방침, 고객센터는 모두 **정적 UI 요소**로 서버 통신이 필요하지 않다.
- 카카오 로그인 버튼은 실제 카카오 OAuth가 아닌 **형식만 유지하는 UI 요소**이며, 실제 인증은 아이디/비밀번호 방식으로 처리된다.
- 회원가입 기능은 신규 사용자 데이터를 생성해야 하므로 API 구현 대상에 포함된다.
