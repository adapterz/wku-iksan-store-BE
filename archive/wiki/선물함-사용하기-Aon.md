# 선물함 사용하기(P_gift_use) 기능 분류

> 전체 프로젝트 흐름 기준: **[1] 상품 선택 → [2] 선물 전송 → [3] 선물 도착 → [4] 선물함에서 사용**
> 본 페이지는 **[4] 단계**(교환권을 열어 바코드를 보여주고 사용 처리)에 해당

---

## ✅ 구현 필요 기능 (API 연동 필요)

| 번호 | 기능 | 해당 단계 | 비고 |
|---|---|---|---|
| 8 | 상품 이미지 | [3]/[4] | 선물함에서 선택한 교환권의 상품 데이터 표시 |
| 9 | 브랜드명 | [3]/[4] | 상품 데이터 필드 |
| 10 | 상품명 | [3]/[4] | 상품 데이터 필드 |
| 11 | 바코드(번호) | [4] | 실제 QR/바코드 스캔 로직은 미구현이나, 화면에 표시되는 **교환권 고유 번호(바코드 숫자)는 해당 선물 데이터에 귀속된 값**이므로 API로 조회 필요. 단, 바코드 이미지 자체는 정적 이미지로 대체 |

---

## ❌ API 구현 불필요 기능 및 근거

| 번호 | 기능 | 근거 | 담당 |
|---|---|---|---|
| 1 | 직전 화면으로 이동 | 순수 클라이언트 라우팅(history.back 등), 데이터 처리 없음 | **FE** |
| 2 | 선물상자로 이동 | 기존 페이지(P_gift_box)로의 라우팅 이동일 뿐, 별도 데이터 처리 없음 | **FE** |
| 3 | 선물하기 홈으로 이동 | 기존 페이지(P_gift_home)로의 라우팅 이동일 뿐, 별도 데이터 처리 없음 | **FE** |
| 4 | 검색 페이지 이동 | 4단계 흐름([1]~[4])에 검색 절차 미정의, 범위 외 기능 | - |
| 5 | 상품 리스트 화면 탭 메뉴 | 상품 탐색/선택은 홈(P_gift_home)의 역할([1] 단계), 사용 페이지 목적과 무관한 중복 네비게이션 | - |
| 6 | 프로필 이미지 | 상태 변화나 데이터 처리가 없는 정적 장식 요소 | - |
| 7 | 사용자명 | 로그인 세션 정보의 단순 표시, 별도 API 로직 불필요 | - |
| 12 | 내 앨범에 저장 | 서버 데이터 처리가 아닌 **디바이스(단말)의 이미지 저장 기능**(네이티브 갤러리 저장 API 호출)이므로 백엔드 API 불필요 | **FE** |

---

## 결론

- 본 페이지의 핵심 기능은 **"선택한 교환권의 상품 정보 + 바코드(번호) 조회"** 로 한정되며, 이 범위에 해당하는 8, 9, 10, 11번만 API 구현 대상입니다.
- 11번 바코드는 **실제 스캔 가능한 QR/바코드 생성 로직은 제외**하되, 화면에 표시할 **바코드 숫자(교환권 고유 값) 데이터는 API로 내려주어야** 하므로 완전한 정적 이미지가 아닌 데이터 기반 표시로 처리합니다.
- 1, 2, 3번(라우팅 이동)과 12번(내 앨범 저장)은 모두 **FE(클라이언트/디바이스) 영역**에서 처리되는 기능으로, 서버 API 구현이 필요하지 않습니다.
- 5, 6, 7번은 이전 페이지(선물함)와 동일한 논리로 **탐색 중복 또는 단순 정적 표시 요소**이므로 API 구현 대상에서 제외됩니다.
