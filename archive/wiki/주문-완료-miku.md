# 주문완료 페이지 - 구현 API 선별

## 구현 O (API 필요, 4개)

| 번호 | 항목 | 이유 |
|---|---|---|
| 5 | 상품 이미지 | 주문한 상품의 이미지 URL을 서버에서 조회해야 표시 가능 |
| 6 | 브랜드명 | 주문 데이터에 포함된 브랜드 정보를 서버에서 조회 |
| 7 | 상품명 | 주문 데이터에 포함된 상품명을 서버에서 조회 |
| 8 | 상품 수량 | 실제 주문 시 확정된 수량 값을 서버에서 조회 |

## 구현 X (API 불필요, FE 라우팅만, 2개)

| 번호 | 항목 | 이유 |
|---|---|---|
| 1 | 선물하기 홈으로 이동 | 페이지 이동(라우팅)만 하면 되므로 별도 데이터 조회 불필요 |
| 10 | 선물함으로 이동 | 페이지 이동(라우팅)만 하면 되므로 별도 데이터 조회 불필요 |

---

## 제외 항목 (4개, 변동 없음)

| 번호 | 항목 | 이유 |
|---|---|---|
| 2 | 외부 링크로 이동 | 배너/광고성 기능, 핵심 흐름과 무관 |
| 3 | 배송지 변경 페이지로 이동 | 선물하기는 모바일 이미지 교환권 방식이라 실물 배송 개념이 약함 |
| 4 | 입력된 배송지 정보 표시 | 3번과 같은 이유로 제외 가능 |
| 9 | 주문내역 페이지로 이동 | 선물함 이동(10번)이 이미 다음 단계로 연결해주므로 우선순위 낮음, 시간 남으면 추가 |

---

## 결론
주문완료 페이지는 **상품 정보(이미지·브랜드명·상품명·수량) 표시는 서버 데이터가 필요해 API 호출이 필요**하지만, **페이지 이동(홈·선물함)은 라우팅만 하면 되므로 FE 로직만으로 구현** 가능하다.
