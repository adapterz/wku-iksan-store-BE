# 2팀 데일리 스크럼 기록 (2026-07-08)

## 1. Git / PR 운영 규칙
- 새 작업할 때마다 브랜치를 재생성해서 PR 요청
- PR 시 **수정한 내용 + 수정 이유**를 구체적으로 작성
- PR 제목도 포괄적이지 않게 구체적으로 작성
- API 명세서 기준 필드명은 **카멜케이스**로 통일

## 2. DB 공유 관련
- 관련 이슈: [#94 DB 공유 방법 문의](https://github.com/adapterz/wku-2026-2-kakao-shop/issues/94)
- 접속 정보(IP, PORT, DB명, 계정, 비밀번호)는 **절대 git에 커밋하지 않음**
- 계정 권한 이슈: IP 허용(방화벽)과 MySQL GRANT는 별개 설정 필요 → 미설정 시 "Access denied" 발생
- 최종 결정: `backend` 계정으로 통일 (Todd가 `GRANT ALL PRIVILEGES ON gift.* TO 'backend'@'%';` 적용)

## 3. PL 미팅
