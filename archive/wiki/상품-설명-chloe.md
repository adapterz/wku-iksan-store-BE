# 상품 상세 페이지 - 구현 API 선별

## 구현 O (API 필요, 3개)

| 번호 | 항목 | 이유 |
|------|------|------|
| 1 | 상품 상세 조회 (GET) | 선택한 상품의 이미지, 상품명, 가격, 브랜드명, 상품 정보를 조회하여 화면에 표시해야 함 |
| 2 | 나에게 선물하기 | 주문(선물) 흐름을 시작하는 기능으로 주문 페이지에서 사용할 상품 정보를 전달하거나 주문서 조회 API와 연동해야 함 |
| 3 | 선물하기 | 선물 주문 흐름을 시작하는 기능으로 주문 페이지에서 사용할 상품 정보를 전달하거나 주문서 조회 API와 연동해야 함 |

---

## 구현 X (API 불필요, FE 상태로만 처리, 13개)

| 번호 | 항목 | 이유 |
|------|------|------|
| 4 | 뒤로가기 | 이전 페이지 이동만 수행 |
| 5 | 검색 버튼 | 검색 UI만 구현 |
| 6 | 공유 버튼 | UI만 구현 |
| 7 | 장바구니 버튼 | UI만 구현 |
| 8 | 별점 표시 | **0점 고정값**으로 UI만 표시하며, 별점 조회 API는 구현하지 않음 |
| 9 | 후기 개수 표시 | **0개 고정값**으로 UI만 표시하며, 후기 조회 API는 구현하지 않음 |
| 10 | 선물후기 탭 | 탭 UI만 구현 |
| 11 | 상품상세 / 선물후기 / 상세정보 탭 | 탭 UI만 구현 |
| 12 | 상품 상세 이미지 | 정적 이미지(UI)만 표시 |
| 13 | 사용방법 영역 | 안내 문구(UI)만 표시 |
| 14 | 좋아요 아이콘 | UI만 구현 |
| 15 | 좋아요 개수 | **0개 고정값**으로 UI만 표시하며, 좋아요 조회 API는 구현하지 않음 |

---

## 결론

상품 상세 페이지는 대부분이 상품 정보 표시와 UI 구성으로 이루어져 있으며, 서버와 통신이 필요한 기능은 **상품 상세 조회(GET)** 와 **나에게 선물하기**, **선물하기**를 통한 주문 흐름 시작이다. 별점, 후기 개수, 좋아요 개수는 모두 **0 고정값**으로 화면에만 표시하며, 후기·좋아요·공유·장바구니 등은 M2 범위에서는 UI만 구현하고 관련 API는 구현하지 않는다.
