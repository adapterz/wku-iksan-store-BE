# 수량 선택 기능 분류

> 전체 프로젝트 흐름은 아래 3단계로 한정됨(범위 기준)
> **[1] 로그인 → [2] 수량 선택 → [3] 주문하기**

---

## ✅ 구현 필요 기능 (API 연동 필요)

| 번호 | 기능 | 해당 단계 | 비고 |
|---|---|---|---|
| 2 | 수량 선택 | [2] | 상품 수량 증가/감소 및 선택 값 관리 |
| 5 | 선물하기 버튼 | [3] | 주문 페이지 이동 및 주문 데이터 전달 |

---

## ❌ API 구현 불필요 기능 및 근거

| 번호 | 기능 | 근거 | 담당 |
|---|---|---|---|
| 1 | 바텀시트 | UI 컴포넌트로만 동작하며 데이터 처리 없음 (프론트엔드에서 상태로 제어) | **FE** |
| 3 | 총 결제금액 | 쿠폰/할인 없이 단순 수량 × 가격 계산으로 프론트에서 처리 가능 | **FE** |
| 4 | 선물담기 | 현재 프로젝트 범위 외 기능이며 UI 디자인만 구현 | **FE** |

---

## 📌 결론

- 전체 흐름은 **로그인 → 수량 선택 → 주문하기**로 구성된다.
- API 구현 대상은 **수량 데이터 처리 및 주문 페이지 이동(데이터 전달 포함)**에 한정된다.
- 바텀시트, 총 결제금액 계산, 선물담기 기능은 모두 서버 통신 없이 동작하는 **프론트엔드 UI 영역**이다.
- 주문하기 기능은 선택된 수량 정보를 기반으로 다음 페이지로 이동하며 주문 데이터를 전달하는 역할을 한다.
