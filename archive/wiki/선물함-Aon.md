# 선물함(P_gift_box) 기능 분류

> 전체 프로젝트 흐름은 아래 4단계로 한정됨(범위 기준)
> **[1] 상품 선택 → [2] 선물 전송 → [3] 선물 도착 → [4] 선물함에서 사용**

---

## ✅ 구현 필요 기능 (API 연동 필요)

| 번호 | 기능 | 해당 단계 | 비고 |
|---|---|---|---|
| 8 | 미사용 / 사용완료 탭 메뉴 | [4] | 교환권 상태(미사용→사용완료) 구분 |
| 9 | 상품 이미지 | [3] | 도착한 선물 데이터 표시 |
| 10 | 브랜드명 | [3] | 상품 데이터 필드 |
| 11 | 상품명 | [3] | 상품 데이터 필드 |
| 12 | 선물한 사람(from) | [2] | 발신자(본인 또는 타 사용자) 정보를 그대로 표시하는 가변 데이터 필드. 자가선물/타인선물 여부와 무관하게 동일 필드로 처리 |
| 13 | 선물한 시간 | [2]/[3] | 선물 전송 시각 데이터 |
| 14 | 사용완료된 상품 목록 | [4] | 사용 처리 결과 상태값 표시 |

## ❌ API 구현 불필요 기능 및 근거

| 번호 | 기능 | 근거 | 담당 |
|---|---|---|---|
| 1 | 직전 화면으로 이동 | 브라우저/앱 네비게이션 스택(history.back 등)을 이용한 순수 클라이언트 라우팅 동작. 데이터 처리 없음 | **FE** |
| 2 | 선물상자(P_gift_box)로 이동 | 현재 페이지 자기 자신으로의 이동 아이콘. 중복 네비게이션이며 별도 기능 없음 | - |
| 3 | 선물하기 홈(P_gift_home)으로 이동 | 기존 페이지로의 클라이언트 라우팅(route push) 동작일 뿐. 이동 후 데이터 조회는 P_gift_home 페이지 자체의 역할 | **FE** |
| 4 | 검색 페이지 이동 | 4단계 흐름([1]~[4])에 상품 검색 절차가 정의되어 있지 않음. 범위 외 기능 | - |
| 5 | 상품 리스트 탭 메뉴(카테고리) | 상품 탐색/선택은 홈(P_gift_home)의 역할([1] 단계). 선물함(받은 선물 확인·사용) 목적과 무관한 중복 네비게이션 | - |
| 6 | 프로필 이미지 | 상태 변화나 데이터 처리가 없는 정적 장식 요소 | - |
| 7 | 브랜드명(사용자 닉네임) | 선물함 소유자(로그인 사용자=받는 사람)의 세션 정보를 단순 표시. 자가/타인 선물 여부와 무관하게 동일하게 고정 표시 | - |

## 결론

- 선물함 페이지는 **"받은 선물 목록 조회(미사용/사용완료 구분) + 사용 처리(바코드 노출 후 상태 변경)"** 기능에 한정되며, 이 범위에 해당하는 항목만 API 구현 대상입니다.
- 검색, 카테고리 탐색, 중복 네비게이션(2번), 장식용 프로필/닉네임 요소(6, 7번)는 핵심 흐름([1]~[4])에 포함되지 않아 API 구현 대상에서 제외됩니다.
- 1번, 3번은 **API 불필요 = 서버 데이터 처리가 필요 없다는 의미**이며, 완전히 구현이 없는 것이 아니라 **FE의 라우팅 로직으로 처리**되는 기능입니다.
- 12번(선물한 사람)은 자가선물/타인선물 여부에 관계없이 **동일한 데이터 필드(from)** 로 처리되므로, 이 구분이 페이지 구조나 API 설계에 영향을 주지 않습니다.
